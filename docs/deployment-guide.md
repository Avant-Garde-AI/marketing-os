# Marketing OS — Deployment Guide

## File Reference

| File | Purpose | Commit to git? |
|------|---------|----------------|
| `.env.cloud` | Template for managed cloud deployment | Yes (no secrets) |
| `.env.self-hosted` | Template for self-hosted deployment | Yes (no secrets) |
| `.env.example.hbs` | Handlebars template used by CLI scaffolding | Yes |
| `.env.local` | Local dev values (generated by CLI) | **No** (gitignored) |
| `.env.production.local` | Production values for Vercel | **No** (gitignored) |
| `vercel.json` | Vercel function config (timeouts, memory) | Yes |

**Rule: never commit files with real credentials.** Use `.env.cloud` or
`.env.self-hosted` as a reference, set actual values in Vercel env vars.

## Cloud Deployment (marketing-os.avant-garde.ai)

### Prerequisites

- Vercel account with the project created
- DNS: `marketing-os.avant-garde.ai` → CNAME `cname.vercel-dns.com`
- Supabase project created
- Shopify Partner Dashboard app created
- GitHub OAuth App created under Avant-Garde-AI org
- Anthropic account with Admin API access

### Step 1: Vercel project setup

```bash
# From the repo root
cd packages/marketing-os/templates/agents
vercel link
```

### Step 2: Set environment variables

In the Vercel dashboard (Settings → Environment Variables), or via CLI:

```bash
# App identity
vercel env add SHOPIFY_APP_URL production        # https://marketing-os.avant-garde.ai
vercel env add NEXT_PUBLIC_APP_URL production     # https://marketing-os.avant-garde.ai

# Shopify
vercel env add SHOPIFY_CLIENT_ID production
vercel env add SHOPIFY_CLIENT_SECRET production
vercel env add SHOPIFY_SCOPES production          # read_products,write_products,...
vercel env add SHOPIFY_EMBEDDED production        # true
vercel env add NEXT_PUBLIC_SHOPIFY_CLIENT_ID production
vercel env add NEXT_PUBLIC_SHOPIFY_EMBEDDED production  # true

# GitHub OAuth
vercel env add GITHUB_CLIENT_ID production
vercel env add GITHUB_CLIENT_SECRET production

# Supabase
vercel env add SUPABASE_URL production
vercel env add SUPABASE_ANON_KEY production
vercel env add SUPABASE_SERVICE_ROLE_KEY production
vercel env add NEXT_PUBLIC_SUPABASE_URL production
vercel env add NEXT_PUBLIC_SUPABASE_ANON_KEY production

# Anthropic
vercel env add ANTHROPIC_API_KEY production
vercel env add ANTHROPIC_ADMIN_KEY production
vercel env add ANTHROPIC_ORG_ID production

# Admin
vercel env add ADMIN_SECRET production            # openssl rand -hex 32
```

### Step 3: Configure Shopify Partner Dashboard

| Field | Value |
|-------|-------|
| App URL | `https://marketing-os.avant-garde.ai` |
| Allowed redirection URL(s) | `https://marketing-os.avant-garde.ai/api/shopify/auth/callback` |
| API scopes | `read_products, write_products, read_orders, read_customers, read_analytics, read_inventory, read_marketing_events` |

### Step 4: Configure GitHub OAuth App

Create at: `github.com/organizations/Avant-Garde-AI/settings/applications/new`

| Field | Value |
|-------|-------|
| Application name | Marketing OS |
| Homepage URL | `https://marketing-os.avant-garde.ai` |
| Authorization callback URL | `https://marketing-os.avant-garde.ai/api/github/auth/callback` |

### Step 5: Run Supabase migrations

```sql
-- Shopify OAuth sessions
create table if not exists shopify_sessions (
  shop text primary key,
  access_token text not null,
  scope text not null default '',
  installed_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
alter table shopify_sessions enable row level security;

-- GitHub connections
create table if not exists github_connections (
  shop text primary key references shopify_sessions(shop),
  github_token text not null,
  github_user text not null,
  scope text not null default '',
  repo_full_name text,
  connected_at timestamptz not null default now(),
  updated_at timestamptz not null default now()
);
alter table github_connections enable row level security;

-- Merchant management
create table if not exists merchants (
  shop text primary key,
  store_name text not null,
  plan text not null default 'free',
  status text not null default 'onboarding',
  shopify_scopes text not null default '',
  github_user text,
  github_repo text,
  anthropic_workspace_id text,
  anthropic_key_id text,
  anthropic_key_hint text,
  agent_runs_this_month integer not null default 0,
  agent_runs_total integer not null default 0,
  installed_at timestamptz not null default now(),
  onboarded_at timestamptz,
  updated_at timestamptz not null default now()
);
alter table merchants enable row level security;

create index idx_merchants_status on merchants(status);
create index idx_merchants_plan on merchants(plan);

-- Helper function for incrementing agent runs
create or replace function increment_agent_runs(shop_domain text)
returns void as $$
  update merchants
  set agent_runs_this_month = agent_runs_this_month + 1,
      agent_runs_total = agent_runs_total + 1,
      updated_at = now()
  where shop = shop_domain;
$$ language sql;
```

### Step 6: Deploy

```bash
vercel --prod
```

### Step 7: Verify

1. Visit `https://marketing-os.avant-garde.ai` — should show login page
2. Visit `https://marketing-os.avant-garde.ai/admin` — should show admin login
3. Test Shopify install: `https://marketing-os.avant-garde.ai/api/shopify/auth?shop=yourstore.myshopify.com`

## Self-Hosted Deployment

### Prerequisites

- Vercel account (free tier works)
- Supabase account (free tier works)
- Shopify Partner account
- GitHub account
- Anthropic API key

### Step 1: Scaffold

```bash
npx marketing-os create my-store
# — or —
npx marketing-os init   # if you already have a theme repo
```

### Step 2: Create your own Shopify Partner App

Go to: `partners.shopify.com` → Apps → Create app

| Field | Value |
|-------|-------|
| App URL | `https://your-vercel-url.vercel.app` |
| Allowed redirection URL(s) | `https://your-vercel-url.vercel.app/api/shopify/auth/callback` |

Copy the Client ID and Client Secret.

### Step 3: Create your own GitHub OAuth App

Go to: `github.com/settings/developers` → New OAuth App

| Field | Value |
|-------|-------|
| Homepage URL | `https://your-vercel-url.vercel.app` |
| Authorization callback URL | `https://your-vercel-url.vercel.app/api/github/auth/callback` |

### Step 4: Create Supabase project

1. Go to `supabase.com` → New project
2. Run the SQL migration above in the SQL Editor
3. Copy project URL, anon key, and service role key

### Step 5: Configure environment

Copy `.env.self-hosted` to `.env.production.local` and fill in your values:

```bash
cp .env.self-hosted .env.production.local
# Edit with your credentials
```

Or set them in Vercel dashboard (Settings → Environment Variables).

### Step 6: Deploy

```bash
cd agents
vercel --prod
```

Update your Shopify Partner App URL and GitHub OAuth callback URL
to match the Vercel deployment URL.

### Step 7: Install on your store

Visit: `https://your-vercel-url.vercel.app/api/shopify/auth?shop=yourstore.myshopify.com`

This kicks off the OAuth flow and installs the app on your store.

## Environment Variable Quick Reference

| Variable | Cloud | Self-Hosted | Where |
|----------|-------|-------------|-------|
| `SHOPIFY_APP_URL` | `https://marketing-os.avant-garde.ai` | Your Vercel URL | Vercel env |
| `NEXT_PUBLIC_APP_URL` | Same as above | Same as above | Vercel env |
| `SHOPIFY_CLIENT_ID` | Your Partner app | Their Partner app | Vercel env |
| `SHOPIFY_CLIENT_SECRET` | Your Partner app | Their Partner app | Vercel env |
| `SHOPIFY_EMBEDDED` | `true` | `true` or `false` | Vercel env |
| `GITHUB_CLIENT_ID` | AG org OAuth app | Their OAuth app | Vercel env |
| `GITHUB_CLIENT_SECRET` | AG org OAuth app | Their OAuth app | Vercel env |
| `SUPABASE_URL` | Shared project | Their project | Vercel env |
| `SUPABASE_SERVICE_ROLE_KEY` | Shared project | Their project | Vercel env |
| `ANTHROPIC_API_KEY` | Your key (shared) | Their key | Vercel env |
| `ANTHROPIC_ADMIN_KEY` | Your admin key | Optional | Vercel env |
| `ANTHROPIC_ORG_ID` | Your org | Optional | Vercel env |
| `ADMIN_SECRET` | Required | Optional | Vercel env |
