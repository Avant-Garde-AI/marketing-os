  Reliable paths: the Vercel **dashboard**, or `vercel env pull` to READ +
  REST `POST /v10/projects/{id}/env` to WRITE (auth token in
  `~/Library/Application Support/com.vercel.cli/auth.json`).
- **⚠ env-pull caveat (learned 2026-07-18):** `vercel env pull` writes
  **sensitive-type** vars (dashboard-added secrets: PENPOT_*, MARKETING_OS_
  API_URL, MOS_PLATFORM_SERVICE_KEY, ANTHROPIC_API_KEY…) as EMPTY strings —
  they are write-only but SET and working in production. Only
  **encrypted-type** vars pull faithfully. Before concluding a var is empty,
  check its `type` via GET `/v10/projects/{id}/env` — an encrypted var that
  pulls empty is REALLY empty (that's how the piped-add victims look).
